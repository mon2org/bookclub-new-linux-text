## 運用ルール

### 1. サイクル
1. **Branch:** 開催回ごとにブランチを作成（例: `session/01`）
2. **Edit:** `docs/` 内の該当する章ファイルを編集
3. **Commit:** 読めた範囲まで記録
4. **Merge:** 最後にこの `README.md` の「しおり」を更新して main にマージ

### 2. ファイルの書き方
以下のテンプレートを使用します。

