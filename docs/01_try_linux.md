# 第01章 Linuxを使ってみよう

## 要点

### Linuxの特徴
|利点|欠点|
|---|---|
|**オープンソース**<br>無償で誰でも自由に利用でき、ソースコードを読んで内部動作を確かめることが可能。|**商用アプリケーションの不足**<br>特にMicrosoft Officeが動作しない点は大きな制約となる。|
|**高品質なソフトウェア**<br />ApacheやMySQLなど、サーバー用途で広く利用される高品質なツールが豊富に揃っている。|**マルチメディア処理の不向き**<br />音楽や動画などの編集作業にはあまり向いていない。|
|**高い信頼性**<br />サーバー用途での豊富な実績があり、安定性が高い。|日本語対応の不備: ソフトウェアやドキュメントが英語のみの場合や、日本語利用で文字化けが発生することがある。|
|**自動化の仕組み**<br />サーバー運用を効率化するためのスクリプトなどの仕組みが充実している。|**優れた学習環境**<br />無償でプログラミング環境を構築でき、OS内部の動作まで学習できる。|

* Linuxカーネル（狭義のLinux）: OSの中核部分であり、ハードウェアの制御などを行う。ユーザーが直接使うツールやアプリケーションは含まれない。
* Linuxディストリビューション（広義のLinux）: Linuxカーネルに加えて、基本的なコマンド群やアプリケーションをパッケージングし、ユーザーがすぐに利用できるようにした形態。一般的に「Linux」という場合、こちらを指す。本書でもこの広義の意味で「Linux」という言葉を使用する。
* 現在主流のディストリビューションは、大きく分けてRed Hat系とDebian系の2系統に分類される。

### 学習環境の構築

効果的な学習のために、実際に手を動かせる環境の用意が強く推奨されている。

* 推奨方法: 仮想化ソフトウェアの利用
  * 既存のOS（ホストOS）上に、仮想的なコンピュータ（仮想マシン）を構築するアプリケーションを利用する方法。
  * 利点: 既存環境を破壊するリスクがなく、誤って環境を壊しても再インストールや復旧が容易。
  * 用語:
    * ホストOS: 仮想化ソフトウェアを動作させているOS（例: Windows, macOS）。
    * ゲストOS: 仮想マシン上で実行されるOS（例: Linux）。
  * 具体的なソフトウェア:
    * Oracle VM VirtualBox: WindowsやIntel Macで利用できる無償の有名なソフトウェア。
    * UTM: Apple Silicon（M1, M2など）搭載のMac向けの仮想化ソフトウェア。
* 読書会: GitHub Codespaces
  * ローカルマシンに仮想環境を構築する代わりに、GitHub Codespacesを利用した。

### CLI（コマンドラインインターフェース）の優位性
|操作|GUI|CUI|
|---|---|---|
|定義|マウス等で画面上のアイコンやメニューをクリックして操作するインターフェース。|キーボードからコマンド（文字列）を入力し、その結果を文字で受け取るインターフェース。|
|ファイルコピー<br>(a.txt を b.txt にコピー)|ファイルアイコンをクリックし、「コピー」メニューを選択後、貼り付け操作を行う。|`cp a.txt b.txt`というコマンドをキーボードから入力し、実行する。|

### CLIを推す4つの理由

1. 作業効率が良くなる
   - GUIでは多数のファイルを一つずつクリックする必要があるが、CLIではワイルドカード（例: A*でAから始まる全ファイル）を用いて一度にまとめてファイルを指定できる。これにより、作業時間が短縮され、見落としなどのミスも防げる。
2. GUIの操作を覚えてもあまり意味がない
   - GUIの見た目や操作方法は数年で大きく変わることが多く、覚えた知識がすぐに古くなる。
   - CLIの基本的なコマンドは10年以上変わらずに使われ続けており、一度学べば長期間役立つ本質的なスキルとなる。
3. プログラムの連携がしやすい
    - Linuxでは「小さなプログラムを部品として連携させる」という思想が根底にある。
    - CLIでは**パイプ（|）**という機能を使って、あるコマンドの出力を別のコマンドの入力に簡単に渡すことができる（例: `grep "ABC" sample.txt | sort`）。
4. 処理の自動化がしやすい
    - 一連のコマンドをファイル（シェルスクリプト）に記述しておくことで、定型的な操作を毎回素早く正確に実行できる。
    - GUIでの繰り返し操作に比べ、ミスなく高速に処理を完了できる。

### ログイン/ログアウト操作
- ログイン
  - システムを利用するユーザーを識別するための認証プロセス。ユーザー名とパスワードを入力してシステムに入室する。
  - ログインが成功すると、コマンド入力を受け付けるプロンプト（例: $ や #）が表示される。
- ログアウト
  - 作業の終了をシステムに伝える操作。「さよならの挨拶」。
  - CLI環境では主に`exit`コマンドを使用する。
  - `logout`コマンドも存在するが、特定の条件下でしか動作しないため、常に`exit`を使うことが推奨。
- シャットダウン
  - コンピュータの電源を切るためにOSを完全に停止させる操作。
  - shutdown コマンドを利用する。このコマンドはシステム全体に影響を及ぼすため、通常は**ルート（root）**と呼ばれる管理者権限を持つスーパーユーザーでしか実行できない。
  - サーバー環境では、シャットダウンはメンテナンスや障害対応など必要最小限の場面でしか行われない、非常に重大な操作であると強調されている。
  - 一般ユーザーが管理者権限でコマンドを実行する際には sudo コマンドが利用される（特にUbuntuなどのディストリビューションで一般的）。
